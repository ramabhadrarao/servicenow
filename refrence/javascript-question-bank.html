Core JavaScript

Q1. Difference between var, let, and const?
  • var: function-scoped, hoisted, re-declarable
  • let: block-scoped, hoisted (TDZ), re-assignable
  • const: block-scoped, must initialize, not re-assigned
Q2. What is hoisting?
Declarations are moved to the top of scope during compilation. var declarations and function declarations hoist; let/const hoist but remain in the temporal dead zone until initialized.
Q3. Explain == vs ===.
== performs type coercion before comparison; === compares both type and value without coercion. Prefer === for predictability.
Q4. What are closures? Give a practical use.
A closure retains access to its lexical scope even after the outer function returns. Use cases: private state, function factories, memoization, event handlers.
Q5. Describe event loop, call stack, and task queues.
Call stack runs synchronous code; Web APIs schedule async work. Macro/microtask queues (e.g., setTimeout, promises) are processed by the event loop when the stack is empty.
Q6. Explain this binding rules.
  • Default: global (window) or undefined in strict mode
  • Implicit: object before dot
  • Explicit: call/apply/bind
  • Arrow functions: lexical this
Q7. Difference between null and undefined?
undefined: variable declared but not assigned; null: intentional absence of value by the programmer.
Q8. What is the difference between map, forEach, and reduce?
  • forEach: iterate for side effects, returns undefined
  • map: transforms each element, returns new array
  • reduce: accumulates into a single value
Q9. What is a promise? Contrast with callbacks.
Promises represent eventual completion/failure of async operations with then/catch; avoid callback hell, standardize chaining, integrate with async/await.
Q10. Explain async/await and error handling.
async functions return promises; await pauses inside async until promise settles. Use try/catch or attach .catch to awaited promise.

ES6+ Features

Q11. What are destructuring and default values?
Extract values from arrays/objects into variables: const {a=1} = obj, const [x=0] = arr. Defaults apply when the value is undefined.
Q12. Difference between rest ... and spread ...?
Rest collects remaining items (function f(a, ...rest)); spread expands iterable/object ([...arr], {...obj}).
Q13. What are modules? How to import/export?
Use export/export default and import. Modules are file-scoped, support tree-shaking, and run in strict mode by default.
Q14. What is optional chaining and nullish coalescing?
?. safely accesses nested properties; ?? provides a default when value is null or undefined.

ServiceNow JavaScript Context

Q15. Client Script types and typical use cases?
  • onLoad: initialize UI
  • onChange: react to field changes
  • onSubmit: validate before submit
Q16. What is g_form? Common methods?
Client-side API for form manipulation: setValue, getValue, setDisplay, addInfoMessage, clearMessages, etc.
Q17. Server-side basics: gs object and GlideRecord?
gs provides logging, utilities. GlideRecord is used to query/update records server-side: new GlideRecord('table'), addQuery, query, next.
