JavaScript History & Core Concepts

üìö History of JavaScript

Understanding JavaScript's evolution helps you grasp why certain patterns exist in ServiceNow development and how the language has shaped modern web applications.

ServiceNow Context

ServiceNow uses JavaScript extensively in both client-side (UI Policies, Client Scripts) and server-side (Business Rules, Script Includes) development. Understanding JavaScript fundamentals is crucial for CSA and CAD certifications.

JavaScript Timeline

1995 - Birth of JavaScript

Brendan Eich created JavaScript in just 10 days at Netscape Communications. Originally called "Mocha", then "LiveScript", finally renamed to "JavaScript" for marketing reasons.

1997 - ECMAScript Standard

JavaScript was standardized as ECMAScript (ES1) by ECMA International to ensure compatibility across different browsers.

2009 - Node.js Revolution

Ryan Dahl created Node.js, bringing JavaScript to server-side development. This concept is similar to how ServiceNow uses JavaScript on the server-side for Business Rules and Script Includes.

2015 - ES6/ES2015

Major update introducing classes, modules, arrow functions, promises, and many other modern features that transformed JavaScript development.

ServiceNow Best Practice

Always use ServiceNow's g_form API instead of direct DOM manipulation. This ensures compatibility across ServiceNow versions and provides better performance.

Control Structures in ServiceNow

Conditional Logic Examples 
// If-else statements for business logic
if (current.priority == '1') {
    current.assignment_group = 'Critical Support Team';
    current.assigned_to = 'senior.engineer';
} else if (current.priority == '2') {
    current.assignment_group = 'Level 2 Support';
} else {
    current.assignment_group = 'Level 1 Support';
}

// Switch statements for multiple conditions
switch (current.category.toString()) {
    case 'hardware':
        current.assignment_group = 'Hardware Team';
        break;
    case 'software':
        current.assignment_group = 'Software Team';
        break;
    case 'network':
        current.assignment_group = 'Network Team';
        break;
    default:
        current.assignment_group = 'General Support';
}

// Loops for processing multiple records
var gr = new GlideRecord('incident');
gr.addQuery('state', 'New');
gr.query();

while (gr.next()) {
    if (gr.priority == '1') {
        // Process critical incidents
        gr.assignment_group = 'Critical Support Team';
        gr.update();
    }
}

üìã ServiceNow JavaScript Environment Differences

Client-Side JavaScript
  • Runs in: User's browser
  • Access to: g_form, g_user APIs
  • Used for: UI Policies, Client Scripts
  • Limitations: No database access
  • Performance: Depends on user's device
Client Script Example 
function onChange(control, oldValue, newValue) {
    if (newValue == '1') { // Critical priority
        g_form.setMandatory('assignment_group', true);
        g_form.addInfoMessage('Critical incident requires immediate assignment');
    }
}
Server-Side JavaScript
  • Runs on: ServiceNow servers
  • Access to: GlideRecord, GlideSystem APIs
  • Used for: Business Rules, Script Includes
  • Capabilities: Full database access
  • Performance: Server-grade processing
Business Rule Example 
// Business Rule - Before Insert
if (current.priority == '1') {
    // Auto-assign critical incidents
    var gr = new GlideRecord('sys_user');
    gr.addQuery('roles', 'CONTAINS', 'itil');
    gr.addQuery('active', true);
    gr.orderBy('last_login_time');
    gr.query();
    
    if (gr.next()) {
        current.assigned_to = gr.sys_id;
    }
}
